# 信任层级

> 按来源可靠性分级审查力度，好钢用在刀刃上

## 概述

不是所有 Skill 都值得全量三层审查。按来源分级，省时省力。5 级信任层级决定了审查深度和默认裁定倾向。

---

## 5 级信任层级表

| 级别 | 来源 | 审查力度 | 默认动作 | 信任依据 |
|:---|:---|:---|:---|:---|
| **T1 官方** | 框架官方团队（OpenClaw 等） | 低 | Layer 1 快审通过即可安装 | GPG 签名 + 官方渠道发布 |
| **T2 高星** | GitHub 仓库 ≥1000 Stars 或下载量 ≥10 万 | 中 | Layer 1 完整审计，通过则安装 | 社区验证 + 长期维护记录 |
| **T3 知名作者** | 历史发布 ≥10 个安全 Skill 且无恶意记录 | 中 | Layer 1 + 依赖审查，通过则安装 | 历史行为可信，但单次不可全信 |
| **T4 新来源** | 新发布者 / 低星仓库 / 未知渠道 | 高 | 完整三层审查，强制行为模拟 | 无历史记录，凭实力过关 |
| **T5 请求凭证** | 任何要求 API Key / Token / 密码输入的 Skill | 最高 | 完整三层 + 人工确认，默认拒绝 | 凭据类 Skill 风险不可逆 |

### 分级说明

- **T1**：仅检查 Skill 是否被篡改（哈希/GPG 校验），跳过深层次审查
- **T2**：执行 30 项静态审计清单，不触发则不进入 Layer 2
- **T3**：在 Layer 1 基础上增加 Layer 3 供应链溯源（依赖审查）
- **T4**：强制走完三层审查，任一环节不通过直接拒绝
- **T5**：无论其他条件如何，一律走 T4 流程 + 要求用户人工确认后才能安装

## 即时拒绝 RED FLAGS（见此即斩）

> 以下 13 条，命中任意一条直接裁定 🚫 恶意，无需走完整三层审查。

| # | RED FLAG | 判定逻辑 | 风险类型 |
|:---|:---|:---|:---|
| 1 | `curl`/`wget` 指向未知 URL | 外发数据到非白名单域名 | R5 |
| 2 | 向外发送数据到外部服务器 | 检测 `POST`/`upload`/`send` + 外部 URL | R5 |
| 3 | 请求凭证 / Token / API Key | SKILL.md 中含 `your API key`、`enter password` | R3 |
| 4 | 读取 `~/.ssh` / `~/.aws` / `~/.config` | 敏感路径 + 读取动词组合 | R3 |
| 5 | 访问 MEMORY.md / USER.md / SOUL.md / IDENTITY.md | 读取身份/记忆文件 | R3/R5 |
| 6 | 使用 `base64` 解码（无正当理由） | `base64 -d` 或 `base64.b64decode()` 出现在非安全审计场景 | R2 |
| 7 | `eval()` / `exec()` 处理外部输入 | AST 检测动态执行 + 非硬编码参数 | R2 |
| 8 | 修改工作目录外的系统文件 | 写入路径不在 `/workspace` 内 | R7 |
| 9 | 安装包但不列出依赖 | 脚本中有 `pip install` 但 `requirements.txt` 缺失或不匹配 | R4 |
| 10 | 网络调用使用 IP 而非域名 | `http://123.456.789.0/` 等 IP 形式 | R5 |
| 11 | 混淆代码（压缩/编码/最小化） | 单行超长字符串、无缩进的压缩代码 | R8 |
| 12 | 请求提权 / sudo 权限 | `sudo`、`root`、`administrator` 关键词 | R6 |
| 13 | 访问浏览器 Cookies / Session | 路径含 `Cookies`、`Session Storage`、`Local Storage` | R3 |

### RED FLAGS 处理流程

```
命中 → 立即阻断 → 不进入评分 → 直接输出 🚫 裁定 → 上报安全情报
```

无需等 Layer 1 跑完，无需等行为模拟。**见即斩。**

## 信任降级规则

以下情况自动降一级，降级后可叠加：

| 触发条件 | 降级 |
|:---|:---|
| Skill 过去一年内有恶意记录 | T1→T4, T2→T4, T3→T4 |
| 单次更新新增网络调用或文件写入 | 降一级 |
| 作者身份无法验证（无 GPG 签名） | T2→T3, T3→T4 |
| 社区有负面反馈（安全相关） | 降一级 |
| 依赖含 Critical CVE | 降一级 |
