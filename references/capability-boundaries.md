# 钟馗能力边界

> Layer 1/2/3 已知盲区与精度上限

## 1. Layer 1 静态审计边界

### 1.1 正则盲区

| 盲区类型 | 示例 | 漏检原因 |
|:---|:---|:---|
| 语义等价绕过 | `fetch(url)` → `urllib.request.urlopen(url)` | 无法穷举所有库 |
| 动态拼接 | `cmd = "rm" + " -rf " + path` | 拼接后字符串不可见 |
| 间接调用 | `getattr(os, 'system')(cmd)` | 无语法解析 |
| 编码绕过 | `\x65\x76\x61\x6c`（eval 的 hex） | 部分覆盖 |
| 注释混淆 | `# 注释\nimport os; os.system(cmd)` | 行内混排可能绕过单行正则 |

### 1.2 语义理解缺失

L1 不做语义分析，无法区分：
- 意图 vs 意外：如"需要提供 API Key"是说明用途还是窃取？
- 良性 vs 恶意 use case：如 `pip install flask` 是正常声明还是恶意依赖？
- 角色扮演 vs 注入：如 `description: "你是安全审计专家"` 是角色设定还是提示注入？

### 1.3 非标准文件格式

`.pyc/.pyo`（编译字节码）/ `.so/.dll`（二进制）/ `.json/.yaml`（仅关键词扫描，不做结构语义）/ 压缩包内文件（不解压）均不审查。

## 2. Layer 2 行为模拟边界

| 能做到 | 做不到 |
|:---|:---|
| 检测给定指令+工具集下的 Agent 决策路径 | 检测内存行为（缓冲区溢出/ROP/shellcode） |
| 识别明显逻辑漏洞（无确认写系统文件） | 检测竞态条件、时序攻击 |
| 覆盖 21 个预定义攻击场景 | 覆盖自定义新攻击向量 |
| 多轮逐步污染检测（≤5轮） | 超 5 轮复杂上下文污染（token 窗口限制） |

精度：误报率 5-10%、漏报率 15-25%。WARN 并非 PASS——确认环节可能被绕过。

## 3. Layer 3 供应链边界

| 数据源 | 更新频率 | 延迟风险 |
|:---|:---|:---|
| CVE 数据库 | 每日 | 新 CVE 最多 24h 延迟 |
| 发布者信誉 | 手动维护 | 新发布者恶意行为可能未收录 |
| 社区反馈 | 实时抓取 | 反馈质量参差 |

无法覆盖：内部镜像投毒（DNS 劫持）/ 构建时注入（CI/CD 篡改）/ 间接依赖 / 二进制依赖。离线时仅覆盖 SBOM/熔断/SDK 等无网维度。

## 4. 跨 Skill 攻击盲区

钟馗只审查单 Skill 目录，不做多 Skill 组合分析：
- 权限串联（A 读 + B 发）
- 信息共享（A 写临时文件 + B 外传）
- 接力注入（A 改 B 配置 → B 执行恶意操作）

无自动化工具可可靠检测跨 Skill 攻击。

## 5. 精度汇总（设计目标，非实测）

| 层面 | 已知模式检出率 | 新型攻击检出率 | 误报率 |
|:---|:---|:---|:---|
| L1 | ~95% | ~40-60% | ~3-5% |
| L2 | ~80-85% | ~50-70% | ~5-10% |
| L3 | ~90%(在线)/~40%(离线) | ~30-50% | ~5% |
