# 供应链溯源（Layer 3）

## 触发条件

- 高风险 Skill（综合评分 < 85）
- 首次发布者（无可验证历史）
- 社区举报触发（>= 3 次安全标记）
- 依赖链含已知高危 CVE

## 十维溯源检查表

| 维度 | 检查内容 | 关键指标 | 扣分 |
|:---|:---|:---|:---|
| 发布者信誉 | GPG 签名验证、历史发布记录、安全事件历史、社区贡献活跃度 | 新发布者（<3 Skill）-> 扣分 | -5 至 -10 |
| 依赖链审计 | 依赖名称与版本偏差、拼写欺诈检测、已知 CVE 交叉比对、依赖深度 | CVE 或拼写欺诈（Levenshtein <= 2）-> 扣分 | -5 至 -8 |
| 版本变更历史 | 版本号规则检查、断崖跳跃检测 | 新增网络调用/文件写入/权限声明 -> 扣分 | -2 至 -8 |
| 社区反馈 | issues/reports/安全标记 | >= 3 次安全标记 -> 自动提升审查等级 | -2 至 -10 |
| SBOM 完整性 | 是否提供标准 SBOM（Software Bill of Materials） | 无 SBOM -> 扣分 | -5 |
| 外部 API 安全评估 | 端点真实性验证、响应签名校验、回调白名单 | 无签名验证 -> 扣分 | -5 |
| SDK 安全准入 | 第三方 SDK 来源、数据采集范围、权限声明 | 无安全评估记录 -> 扣分 | -3 至 -5 |
| 熔断与降级预案 | 依赖服务不可用时的安全降级策略 | 无降级声明 -> 扣分 | -3 |
| 外部信息源信誉 | 搜索引擎/API/URL 域名声誉与安全记录 | 域名仿冒且无威胁情报校验 -> 扣分 | -5 至 -10 |
| 持久化写入审计 | 是否写入定时任务/系统配置/长期记忆 | 写入持久化存储但未校验 -> 扣分 | -5 至 -10 |

## L3 扣分汇总

| 风险等级 | 扣分范围 | 典型场景 |
|:---|:---|:---|
| 高风险 | -25 至 -30 | 依赖含多个 CVE + 无 SBOM + 发布者信誉缺失 |
| 中风险 | -15 至 -24 | 依赖含一个高危 CVE + 部分维度缺失 |
| 低风险 | -5 至 -14 | 仅个别维度扣分 |

## 依赖项运行时行为监控

| 监控指标 | 检测方法 | 风险标记 |
|:---|:---|:---|
| 运行时网络出站 | 监控 DNS/HTTP/HTTPS 连接，比对威胁情报库 | dep_runtime_network |
| 文件系统异常写入 | 检测对非工作目录/系统路径的写操作 | dep_anomalous_fs_write |
| 子进程 spawn | 监控 subprocess/os.system/fork+exec 进程链 | dep_process_spawn |

## 发布者信誉动态评分模型

| 条件 | 信誉分变动 | 触发 |
|:---|:---|:---|
| 10+ Skill 无安全事件 | +5 | 每次审查更新 |
| 历史安全审查通过率 > 95% | +3 | 每次审查更新 |
| 多个 Skill 被标记可疑（>= 3） | -10 | 实时触发 |
| 社区安全事件报告 | -15 | 实时触发 |
| 首次发布 Skill | 初始分 50 | 新发布者默认 |
