# 评分与裁定规则

> 钟馗采用加权扣分制 + 一票否决机制，输出绿/黄/红三标裁定。

## 评分公式

钟馗采用双路径权重：**完整路径**（Layer 2 行为模拟产出可量化分数时启用）与**离线默认路径**（Layer 2 未运行时启用，权重自动重分配）。两条路径均与一票否决/红线即时拒绝机制短路叠加。

### 路径 A：完整路径（L1 + L2 + 数据安全 - L3 罚分）

```
总分 = Layer1_Score * 0.35 + Layer2_Score * 0.55 + DataSecurity_Score * 0.10 - Layer3_Penalty
```

- 触发条件：调用 `Scorer.compute()` 时传入 `l2_score` 非 None（即 Layer 2 行为模拟已运行并产出 0-100 量化分数）。
- 适用场景：在线深度审查模式，三层数据齐备。

### 路径 B：离线默认路径（L1 + 数据安全 - L3 罚分）—— 当前 zhongkui.py 实际运行路径

```
总分 = Layer1_Score * 0.85 + DataSecurity_Score * 0.15 - Layer3_Penalty
```

- 触发条件：调用 `Scorer.compute()` 时 `l2_score` 未传入或为 None（即 Layer 2 静态审计未运行，或未产出可量化分数）。
- 适用场景：`zhongkui.py` 默认调用 `scorer.compute(auditor, l3_penalty=0)` 未传 `l2_score`，即走此路径。
- 权重重分配：W1 由 0.35 提升至 0.85，W3 由 0.10 提升至 0.15，W2 缺省置 0；总分仍归一化至 0-100 区间。

### 组件权重对照

| 权重 | 完整路径 (A) | 离线默认路径 (B) | 组件 | 说明 |
|:---|:---|:---|:---|:---|
| W1 | 0.35 | 0.85 | Layer 1 静态审计 | 满分 100，由 54 项检查逐项扣分得出 |
| W2 | 0.55 | 0.00（不参与） | Layer 2 行为模拟 | 映射到 0-100；缺省时权重并入 W1/W3 |
| W3 | 0.10 | 0.15 | 数据安全声明 | 满分 100（D1-D7） |
| Penalty | 0-30 | 0-30 | Layer 3 扣分 | 累加制；离线默认路径下未传入时为 0 |

> 实际权重由 `core/scorer.py:compute()` 中 `l2_score is not None` 判定分支决定，与本节文档对齐。
>
> 说明：W1 行的"54 项检查" = M(7)+C(17)+S(12)+P(11)+D(7)，含一票否决项（C1/C3/C4/C5/C7/S6）；与 `core/auditor.py` `run_full()` 注释（M1-M7 / C1-C17 / S1-S12 / P1-P11 / D1-D7）一致。

### 场景化权重调整（仅适用于完整路径 A）

| 场景类型 | W3 调整 | 说明 |
|:---|:---|:---|
| 金融/医疗/政务 | 0.15-0.20 | 数据安全权重提升 |
| 纯工具 | 0.05 | 数据安全权重降低 |
| 默认 | 0.10 | — |

权重变更须在审查报告中显式标注。

## 裁定阈值

| 总分 | 裁定 | 处理建议 |
|:---|:---|:---|
| >= 85 | ✅ 干净 | 可安全安装 |
| 70-84 | ⚠️-low | 低风险可疑，附带主要风险点摘要，建议人工复核 |
| 60-69 | ⚠️-high | 高危可疑，建议拒绝安装 + 详细风险报告 |
| < 60 | 🚫 恶意/高风险 | 禁止安装，上报安全情报 |

---

## 一票否决（5 项）

命中任一项 -> 直接裁定 🚫 恶意，跳过评分计算：

| 编号 | 否决原因 |
|:---|:---|
| R1 | 提示注入 |
| R2 | 恶意代码执行 |
| R3 | 凭证窃取 |
| R5 | 数据外传 |
| R7 | 持久化后门 |

---

## Layer 3 扣分规则

| 扣分值 | 触发条件 |
|:---|:---|
| -2 | 依赖含低危 CVE（info/low） |
| -5 | 新发布者（< 3 个 Skill，无可验证安全记录） |
| -8 | 依赖含高危 CVE（critical/high）或安全事件报告 |
| -10 | 无 SBOM 且无任何可验证安全记录 |

累计扣分上限 30 分。

---

## 多引擎仲裁（扫描器一致性增强）

| 规则 | 条件 | 裁定 |
|:---|:---|:---|
| 双绿 | 两引擎均 >= 85 | ✅ 绿标 |
| 绿+黄 | 一绿一黄（任一 70-84） | ⚠️-low |
| 绿+红 | 一绿一红（任一 < 60） | ⚠️-low（分歧升级至黄标底限） |
| 双红 | 两引擎均 < 60 | 🚫 红标 |

分歧合并规则：任一引擎判定红标 -> 最终裁定不低于黄标。

---

## 退出码定义

| 退出码 | 含义 |
|:---|:---|
| 0 | 正常完成（Skill 安全或可疑但不危险） |
| 1 | 参数错误或目标路径不存在 |
| 2 | 引擎内部错误（未捕获异常，非审计结果） |
| 3 | 一票否决或红线命中（Skill 恶意） |

```
置信度 = (1 - 加权误报率) * prior_trust + fraud_pattern_detection_score * 0.5
```

| 阈值 | 裁定 |
|:---|:---|

| 70-89 | 中置信可疑 |
