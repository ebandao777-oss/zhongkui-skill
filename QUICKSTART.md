# 钟馗.Skill 快速上手

> 直来直去，三步上手。

## 安装

把 `zhongkui-skill/` 目录放入 skills 目录即可。

## 基本用法

### 场景 1：首次审查一个陌生 Skill

```
审 E:\skills\downloaded-skill
```

执行完整三层审查。审查完成后你会看到结构化报告：

```
## 钟馗裁定：[✅]
**总分**：92 / 100
**一句话**：未发现安全问题，通过54项静态检查

### 扣分项
（空）

### 审计覆盖
- 文件数：3（SKILL.md, install.py, README.md）
- 检查项：54（命中 0，通过 54）
```

### 场景 2：审查结果发现扣分——下一步怎么做

假设你看到：

```
## 钟馗裁定：[⚠️]
**总分**：72.5 / 100
**一句话**：发现 3 个安全问题，无高危项

### 扣分项
| 层级 | 检查项 | 风险类型 | 扣分 | 命中内容（行号） |
|:---|:---|:---|:---|:---|
| L1 | C3 | R3 | -8 | SKILL.md:L45 |
| L1 | M4 | R6 | -5 | SKILL.md:L0 |
| L1 | D1 | R9 | -2 | SKILL.md:L0 |
```

**应对步骤**：
1. 看扣分项的风险类型 → R3 是凭证窃取，需要认真看
2. 打开 SKILL.md 第 45 行 → 看到 `请提供 API_KEY` → 确认是否合理声明了用途
3. D1 是数据安全声明缺失 → 加上 `## 数据安全` 声明即可
4. 修复后重新 `审` 一遍，确认分数回升

### 场景 3：快速初筛——我有 10 个 Skill 要装

```
快审 E:\skills\repo\skill-01
快审 E:\skills\repo\skill-02
...
```

快审只跑 Layer 1（秒级），适合批量过筛。快审通过 ✅ 的直接装，⚠️/🚫 的再走完整审查。

### 场景 4：我只关心这个 Skill 的依赖有没有问题

```
审依赖 E:\skills\new-skill
```

跳过 L1 扫描和 L2 行为模拟，只看 10 维供应链溯源——发布者信誉、CVE 依赖、SBOM 等。适合已人工看过 SKILL.md 但想确认依赖安全的场景。

### 场景 5：Layer 1 通过但怀疑有隐藏指令

```
审行为 E:\skills\suspicious-skill
```

只跑 L2 行为模拟（21 场景），做专项对抗测试。适合"SKILL.md 看起来干净但感觉不对"的场景。

### 直接运行脚本

```bash
# 完整审计
python zhongkui.py E:\skills\some-skill

# 快速审计（仅 Layer 1）
python zhongkui.py E:\skills\some-skill --quick
```

## 输出解读

| 裁定 | 含义 | 行动 |
|:---|:---|:---|
| ✅ 干净 | ≥ 85 分 | 放心装 |
| ⚠️-low | 70-84 分 | 低风险可疑，看一眼扣分项再决定 |
| ⚠️-high | 60-69 分 | 高危可疑，建议拒绝 |
| 🚫 恶意 | < 60 或命中否决 | 别装，上报 |

评分由三层加权计算：Layer 1 静态审计（W1=0.35） + Layer 2 行为模拟（W2=0.55） + 数据安全声明（W3=0.10），Layer 3 额外扣分 0-30。红队对抗测试通过率 < 95% 自动阻断 [规划中，尚未实现]。

## 常见问题

**Q: 为什么我的 Skill 被判 R8？**
A: R8 检查隐蔽 Unicode 字符（零宽空格/方向控制字符）和条件触发逻辑。八成是复制粘贴时带入了不可见字符。用 `审` 命令会告诉你具体行号，删掉那段重写即可。

**Q: 审查多久？**
A: Layer 1 秒级（纯正则扫描）。Layer 2 约 30 秒（21 场景模拟推理）。Layer 3 取决于依赖数量和网络状况，完全离线时跳过网络维度。

**Q: 脚本和对话审查有什么区别？**
A: 脚本（`python zhongkui.py`）只执行 Layer 1 静态审计；对话中使用 `审` 可执行完整三层审查（含 L2 行为模拟和 L3 供应链溯源）。

**Q: 黄标为什么有 high/low 之分？**
A: 60-69 分为高危可疑（⚠️-high），建议拒绝；70-84 分为低风险可疑（⚠️-low），附带主要风险点供你判断。

**Q: 误报怎么办？**
A: 记下扣分项的具体内容，反馈给钟馗。多引擎仲裁机制保证不会单一规则误杀。

**Q: 审完后文件编码报错怎么办？**
A: 钟馗会自动尝试 UTF-8 → GBK → Latin-1 降级读取。如果都失败，会在报告中明确标注"文件编码失败"并跳过该文件，不会静默忽略。

**Q: 审查中断了怎么办？**
A: 钟馗遇到非致命错误（如某文件无法读取、某网络请求超时）会主动报告异常并继续其余检查。如果完全中断，检查目标路径是否正确、是否有权限。

**Q: 能审多个 Skill 吗？**
A: 目前单次审查只支持一个 Skill 目录。批量审查请逐个 `快审` 或写一个脚本循环调用 `python zhongkui.py`。

**Q: 钟馗审不了什么？**
A: 深度运行时行为（内存注入/进程劫持）、跨 Skill 组合攻击、多层嵌套加密载荷、全新零日攻击向量。详见 SKILL.md 的「能力边界」章节。
